Entendendo o Monitoramento

Como funciona o diagnóstico operacional de filiais

Como funciona o monitoramento?

O monitor compara continuamente as vendas de Hoje com a curva histórica da própria filial, baseada na média de Ontem e de 7 Dias Atrás. A comparação é feita minuto a minuto (proporcional ao horário decorrido do dia) para evitar falsos positivos.

O que significam os Status?

  • ONLINE A filial está operando e faturando dentro do ritmo de vendas esperado para o horário.
  • OFFLINE A filial possui histórico relevante de vendas online, mas registrou zero vendas hoje, embora o esperado para o horário seja de pelo menos 1.2 pedidos. Sinaliza possível perda de sinal do PDV, falha de API ou loja fechada.
  • CRÍTICO A filial já vendeu hoje, mas está há mais de 2 horas (e pelo menos 3x seu intervalo normal) sem faturar nada. Indica uma interrupção repentina de comunicação durante a operação.
  • ALERTA Silêncio de vendas acima de 1 hora, ou faturamento acumulado hoje abaixo de 40% da curva de referência.
  • INATIVA Filiais com histórico extremamente baixo de vendas online (vendas ocasionais). Não acionam alertas automáticos.

Frequência de Sincronização

O servidor faz uma varredura automática na API VTEX a cada 20 minutos buscando novos pedidos. O painel também se atualiza automaticamente a cada 60 segundos.

Como evitamos Alarmes Falsos?

Para evitar alertas falsos decorrentes do intervalo de sincronização (efeito dente de serra), o monitor calcula o ócio das filiais utilizando o horário da última sincronização bem-sucedida com a VTEX como referência, e não o relógio atual do seu computador. Se a sincronização falhar ou atrasar por mais de 45 minutos, o sistema passa a utilizar o horário real para alertar que os dados estão desatualizados.
